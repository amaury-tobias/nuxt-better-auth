This guide is for **Nuxt module authors** who want to integrate with Nuxt Better Auth. If you're building an application, see the [Configuration](/getting-started/configuration) guide instead.

## When to Use These Hooks

Use these hooks when your module needs to:
- Add authentication to module-provided routes
- Extend the Better Auth configuration
- Access the auth instance for custom functionality

## Access User Sessions

Other modules can use server utils (auto-imported in Nitro handlers):

```ts
const { user, session } = await getUserSession(event)
if (!user) throw createError({ statusCode: 401 })
```

## Extend Config via Hook

Use `better-auth:config:extend` to add plugins or hooks:

```ts
// In your Nuxt module
export default defineNuxtModule({
  setup(options, nuxt) {
    nuxt.hook('better-auth:config:extend', (config) => {
      config.plugins = [...(config.plugins || []), myPlugin()]
    })
  }
})
```

## Restrict Email Domains

Use `databaseHooks.user.create.before` to validate domains:

```ts
// auth.config.ts
export default defineServerAuth(() => ({
  databaseHooks: {
    user: {
      create: {
        before: async (user) => {
          const domain = (user.email as string).split('@')[1]
          if (!['company.com', 'partner.org'].includes(domain)) {
            return false // Reject signup
          }
        }
      }
    }
  }
}))
```

## Auth Lifecycle Events

React to signup, login, logout:

```ts
export default defineServerAuth(() => ({
  databaseHooks: {
    user: {
      create: { after: async (user) => { /* new user created */ } }
    },
    session: {
      create: { after: async (session) => { /* user logged in */ } },
      delete: { after: async (session) => { /* user logged out */ } }
    }
  }
}))
```
