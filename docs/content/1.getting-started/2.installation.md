## Prerequisites

- Nuxt v3.10+

## Add to project

::steps

### Install the module

::code-group{sync="pm"}
```bash [pnpm]
pnpm add @onmax/nuxt-better-auth better-auth
```
```bash [npm]
npm install @onmax/nuxt-better-auth better-auth
```
```bash [yarn]
yarn add @onmax/nuxt-better-auth better-auth
```
::

::tip
You can also run `npx nuxi module add @onmax/nuxt-better-auth` to auto-inject the module into `nuxt.config.ts`.
::

::warning
**pnpm users**: Add `auto-install-peers=true` to `.npmrc` or install manually: `pnpm add better-auth`
::

### Set Environment Variables

Add these environment variables to `.env`:

1. **Secret Key**

A secret value used for encryption and hashing. It must be at least 32 characters and generated with high entropy. Click the button below to generate one. You can also use `openssl rand -base64 32` to generate one.

```txt [.env]
NUXT_BETTER_AUTH_SECRET=
```

::generate-secret
::

Or generate via terminal:

```bash
openssl rand -base64 32
```

2. **Base URL**

Required for production deployments. In development, the module uses `http://localhost:3000` automatically.

```txt [.env]
NUXT_PUBLIC_SITE_URL=https://your-domain.com
```

### Configure Nuxt

Add the module to your `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@onmax/nuxt-better-auth'],
})
```

### Create Configuration Files

Two configuration files connect Nuxt with Better Auth.

1. **Server Configuration**: `auth.config.ts` (project root)
2. **Client Configuration**: `client-auth.config.ts` (project root, or `app/` for Nuxt 4)

Create these two configuration files:

```ts [auth.config.ts]
import { defineServerAuth } from '@onmax/nuxt-better-auth'

export default defineServerAuth(() => ({
  emailAndPassword: { enabled: true }
}))
```

```ts [client-auth.config.ts]
import { createAuthClient } from 'better-auth/vue'

export function createAppAuthClient(baseURL: string) {
  return createAuthClient({ baseURL })
}
```
